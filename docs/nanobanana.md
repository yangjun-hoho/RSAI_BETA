**나노 바나나**는 제미니의 자체 이미지 생성 기능의 이름입니다.
Gemini는 대화형으로 이미지를 생성하고 처리할 수 있습니다.
텍스트, 이미지 또는 둘 다를 사용하여 만들 수 있습니다. 이를 통해 생성, 편집 및 수정할 수 있습니다.
전례 없는 제어력으로 시각적 요소를 반복적으로 수정하세요.

나노 바나나는 Gemini API에서 사용할 수 있는 두 가지 모델을 지칭합니다.

- **나노 바나나 2**: [Gemini 3.1 플래시 이미지 미리보기](https://ai.google.dev/gemini-api/docs/models/gemini-3.1-flash-image-preview) 모델(`gemini-3.1-flash-image-preview`)을 사용합니다. 이 모델은 Gemini 3 Pro 이미지의 고효율 버전으로, 속도와 대용량 개발자 사용 사례에 최적화되어 있습니다.
- **나노 바나나 프로**: [제미니 3 프로 이미지 미리보기](https://ai.google.dev/gemini-api/docs/models/gemini-3-pro-image-preview) 모델(`gemini-3-pro-image-preview`)을 사용합니다. 이 모델은 전문적인 에셋 제작을 위해 설계되었으며, 고급 추론("사고") 기능을 활용하여 복잡한 지침을 따르고 고품질 텍스트를 렌더링합니다.
- **나노 바나나**: [Gemini 2.5 Flash Image](https://ai.google.dev/gemini-api/docs/models/gemini-2.5-flash-image) 모델(`gemini-2.5-flash-image`). 이 모델은 속도와 효율성을 위해 설계되었으며, 대용량 저지연 작업에 최적화되어 있습니다.

생성된 모든 이미지에는 [SynthID 워터마크](https://ai.google.dev/responsible/docs/safeguards/synthid)가 포함됩니다.

## Gemini 3 이미지 모델의 새로운 기능

Gemini 3는 최첨단 이미지 생성 및 편집 모델을 제공합니다. Gemini 3.1
Flash Image는 속도와 대용량 사용 사례에 최적화되어 있으며 Gemini 3도 마찬가지입니다.
Pro Image는 전문적인 에셋 제작에 최적화되어 있습니다.
고급 추론을 통해 가장 까다로운 워크플로우를 처리하도록 설계되었습니다.
그들은 복잡하고 여러 단계를 거치는 생성 및 수정 작업에 탁월합니다.

- **고해상도 출력** : 1K, 2K, 4K 영상 생성 기능이 내장되어 있습니다.
  - **Gemini 3.1 플래시 이미지**는 더 작은 512픽셀(0.5K) 해상도를 추가합니다.
- **고급 텍스트 렌더링**: 인포그래픽, 메뉴, 다이어그램 및 마케팅 자료에 사용할 수 있는 가독성이 뛰어나고 스타일리시한 텍스트를 생성할 수 있습니다.
- **구글 검색을 활용한 검증**: 이 모델은 구글 검색을 도구로 사용하여 사실을 검증하고 실시간 데이터(예: 현재 날씨 지도, 주식 차트, 최근 이벤트)를 기반으로 이미지를 생성할 수 있습니다.
  - **Gemini 3.1 Flash Image**는 웹 검색과 함께 Google 이미지 검색 그라운딩 기능을 통합합니다.
- **사고 모드**: 이 모델은 "사고" 과정을 활용하여 복잡한 질문에 대한 추론을 진행합니다. 최종 고품질 결과물을 생성하기 전에 구성을 다듬기 위해 중간 "사고 이미지"(백엔드에서 확인 가능하지만 비용은 발생하지 않음)를 생성합니다.
- **최대 14개의 참조 이미지**: 이제 최대 14개의 참조 이미지를 조합하여 최종 이미지를 만들 수 있습니다.
- **새로운 화면비** : Gemini 3.1 플래시 이미지 미리보기에 1:4, 4:1, 1:8, 8:1 화면비가 추가되었습니다(https://ai.google.dev/gemini-api/docs/image-generation#aspect_ratios_and_image_size).

**표시 요구 사항**

Grounding에서 Google 검색을 이용한 이미지 검색을 사용할 때는 다음 사항을 준수해야 합니다.
다음과 같은 조건을 충족하는 경우:

- **출처 표기**: 사용자가 링크로 인식할 수 있는 방식으로 원본 이미지가 포함된 웹페이지(이미지 파일 자체가 아닌 "이미지가 포함된 페이지")에 대한 링크를 제공해야 합니다.
- **직접 탐색**: 원본 이미지를 함께 표시하기로 선택한 경우, 원본 이미지에서 해당 원본 웹페이지로 바로 이동할 수 있는 단일 클릭 경로를 제공해야 합니다. 여러 번 클릭해야 하는 경로 또는 중간 이미지 뷰어 사용 등 최종 사용자가 원본 웹페이지에 접근하는 과정을 지연시키거나 추상화하는 모든 구현 방식은 허용되지 않습니다.

**응답**

이미지 검색을 활용한 근거 있는 답변의 경우, API는 명확한 출처 표기를 제공합니다.
또한 출력 결과를 검증된 출처와 연결하기 위한 메타데이터도 포함됩니다. 주요 필드는 다음과 같습니다.
`groundingMetadata` 객체 포함:

- **`imageSearchQueries`**: 모델이 시각적 컨텍스트(이미지 검색)를 위해 사용하는 특정 쿼리입니다.
- **`groundingChunks`**: 검색된 결과에 대한 소스 정보를 포함합니다.
  이미지 소스의 경우, 새 이미지를 사용하는 리디렉션 URL로 반환됩니다.
  청크 유형. 이 청크에는 다음이 포함됩니다.

  - **`uri`**: 어트리뷰션에 사용할 웹페이지 URL(방문 페이지).
  - **`image_uri`**: 이미지의 직접 URL입니다.
- **`groundingSupports`**: 생성된 항목을 연결하는 특정 매핑을 제공합니다.
  콘텐츠를 해당 인용 출처에 연결합니다.

- **`searchEntryPoint`**: "Google 검색" 칩이 포함되어 있습니다.
  검색 제안을 표시하기 위해 HTML 및 CSS 규정을 준수합니다.

### 최대 4K 해상도의 이미지 생성

제미니 3 이미지 모델은 기본적으로 1K 이미지를 생성하지만 2K 이미지도 출력할 수 있습니다.

#### 생각의 특징

생각 서명은 암호화된 표현입니다.
모델의 내부 사고 과정을 나타내며 추론 맥락을 보존하는 데 사용됩니다.

응답을 보낼 때는 받은 그대로 다시 보내야 합니다.
다음 차례에 대화 내역이 표시됩니다. 생각의 흔적을 공유하는 데 실패했습니다.
응답이 실패할 수 있습니다. [thought signature](https://ai.google.dev/gemini-api/docs/thought-signatures)를 확인하세요.
서명에 대한 자세한 설명은 관련 문서를 참조하십시오.

사고 서명은 다음과 같이 작동합니다.

- 응답에 포함된 이미지 `mimetype`을 가진 모든 `inline_data` 부분에는 서명이 있어야 합니다.

- 모델은 사용자가 명시적으로 요청한 이미지 출력 개수를 항상 정확하게 따르지는 않습니다.

  구글 검색을 활용한 실질적인 사고 과정, 즉 기본 "사고" 과정
  이미지 생성 전에 구성을 정밀하게 조정하며, 최대 4K 해상도의 이미지를 생성할 수 있습니다.
  해상도에 대한 자세한 내용은 모델 [가격](https://ai.google.dev/gemini-api/docs/pricing#gemini-3-pro-image-preview) 및 [기능](https://ai.google.dev/gemini-api/docs/models/gemini-3-pro-image-preview) 페이지를 참조하세요.
  세부.

- **제미니 2.5 플래시 이미지(나노 바나나)**는 속도를 위해 설계되었습니다.
  효율성. 이 모델은 대용량, 저지연 작업에 최적화되어 있습니다.
  1024px 해상도로 이미지를 생성합니다. 모델 [가격](https://ai.google.dev/gemini-api/docs/pricing#gemini-2.5-flash-image)을 확인하세요.
  자세한 내용은 [기능](https://ai.google.dev/gemini-api/docs/models/gemini-2.5-flash-image) 페이지를 참조하세요.
  세부.
